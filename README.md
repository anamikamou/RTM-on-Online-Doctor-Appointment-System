# RTM-for-Online-Doctor-Appointment-System
a small project where Requirement Traceability Matrix for a doctor appointment system is designed

## Project overview
This project demonstrates a simple Requirement Traceability Matrix (RTM) for a Doctor Appointment System. It maps and covers all the functional requirements to corresponding test cases to ensure complete user details and requirement validation check.
## Definition of RTM
RTM stands for Requirements Traceability Matrix; it is a documentation process where all the requirements must match with the test cases so that nothing gets missed to validate. There are 3 types of Requirement Traceability Matrix (RTM):
1. Forward traceability matrix
2. Backward traceability matrix
3. Bi-directional traceability matrix
## The importance of RTM
1.	Each test cases properly tested and documented, so that there is no chance to miss any requirements from user end.
2.	The matrix can enlighten the tester if there any newly added requirements affect the previous requirements
3.	RTM helps in assessing the effect on the QA team to reproduce the test case.
4.	It is necessary to update the RTM whenever there is a change in requirement.
